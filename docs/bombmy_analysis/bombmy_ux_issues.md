# 幫賣後台 (Bombmy Dashboard) UX 觀察與問題清單

基於頁面巡覽與視覺截圖，以下是系統可優化或調整的 UX 觀察：

## 1. 狀態與標題不一致 (嚴重)
在「我的影片」與「我的商品」的列表頁中，上方的大標題竟然顯示為「新增影片」與「新增商品」。這違背了使用者的預期，列表頁就應該叫列表頁標題（如：管理影片、影片列表），「新增」應僅作為按鈕。這會讓使用者困惑目前到底是在新增模式還是列表模式。

## 2. 資訊密度與版面配置過於空曠
首頁「直播分析」的四張圖表卡片非常大，但在無資料 (`尚無資料`) 時留白過多，導致整個頁面顯得極度空洞。
**建議**：空狀態下可給予佔位圖 (Placeholder) 或導引操作 (例如：「去建立你的第一場直播」)。

## 3. 按鈕與搜尋列配置不直觀
- 在「虛擬使用者」頁面，頂部搜尋列右側的藍底按鈕寫的是「搜尋」，而新增使用者的操作則變成網格內第一張卡片的 `+` 號。但在其他頁面（如商品、影片），搜尋列右邊的藍色按鈕是「新增」。
- 這導致跨頁面的操作慣性 (Mental Model) 被破壞，使用者需要重新學習每個頁面的新增按鈕在哪裡。

## 4. 編輯與明細操作流程設計
「虛擬使用者」採用了左側網格列表、右側表單的 `Master-Detail` 結構。但在預設狀態下，右側只顯示「請先選擇使用者」。
**建議**：在桌面版這樣的設計雖然省跳轉，但網格寬度會被壓縮。若卡片資訊增加，建議統一改回列表式 + 點擊進入獨立編輯頁，或是改用 Drawer (側邊抽屜) 來做輕量編輯。

## 5. 安全性設定未被突顯
在「廠商設定」中，「登出所有裝置」這個帶有高風險與重大操作意義的按鈕，採用紅色滿版設計，並直接塞在 Pixel ID 等設定欄位的下方。
**建議**：這類按鈕應獨立拉出一個「安全性區塊」，或置於更外層的帳號設定選單中，避免誤觸。

## 6. 頁籤切換缺乏動態回饋
大多數列表頁的過濾或分頁狀態（如扣點紀錄的 日期/月份/年份 區間），採用按鈕樣式呈現。切換時若能帶入更明確的 Tab 樣式與過渡動畫 (Micro-animations)，整體質感將大幅提升。

## 7. 重建優先頁面建議 (UI 翻新)
若要進行 UI 重建或優化，強烈建議從以下頁面著手：
1. **直播分析 (Dashboard)**：這是門面，必須將空狀態與圖表呈現做得更 Premium (例如加入半透明毛玻璃效果或更豐富的漸層色)。
2. **廠商設定 (表單頁)**：目前表單過於平淡，可加入分類卡片、Toggle Switches 取代部分傳統輸入，提升專業感。
