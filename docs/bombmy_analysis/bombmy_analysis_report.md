# 幫賣後台 (Bombmy Dashboard) 結構化分析報告

## 1. 直播分析
### 頁面名稱
直播分析 (Dashboard)
### URL / 路由
`/dashboard` (推測為預設進入點)
### 頁面類型
儀表板 (Dashboard)
### 主要用途
提供直播相關之全站數據總覽，包含觀看人數、互動與銷售報表。
### 主要區塊
1. 頂部 KPI 卡片區（總觀看人數、本站報名/參與、聯盟報名/參與）
2. 圖表區（同時在線人數、廣告推薦人、互動數與時間點、銷售報表）
### 主要按鈕與操作
- `真實數據` Toggle 切換開關
- `查詢直播時數` 按鈕 (點擊會開啟 Modal)
### 欄位與表單元素
- (Modal 內) 開始時間、結束時間 (日期選擇器)
### 特殊元件
- KPI 數據顯示卡片
- 四大數據圖表 (Chart)
### 截圖檔名
`01_dashboard.png`
### 流程備註
預設狀態下圖表無資料時顯示為「尚無資料」。
### 問題點 / 疑問點
- 四個圖表的區塊佔用空間極大，但空狀態下顯得較為空洞。
- 頂部藍色區塊寫著「選擇直播 / 尚未選取」，似乎是個下拉選單但設計像按鈕。
### 是否屬於 MVP 必做
是

## 7. 虛擬使用者
### 頁面名稱
編輯虛擬使用者
### URL / 路由
`/virtual-users` (推測)
### 頁面類型
列表 / 設定明細 (Master-Detail)
### 主要用途
建立虛擬人物 (假帳號)，供直播或留言組使用，營造互動熱絡感。
### 主要區塊
- 左側：虛擬使用者網格列表
- 右側：編輯表單區 (預設為空，提示「請先選擇使用者」)
### 主要按鈕與操作
- 卡片首個項目為 `+` 新增按鈕
- 點擊卡片以在右側編輯
- `搜尋` 按鈕 (藍底)
### 欄位與表單元素
- 搜尋輸入框
### 特殊元件
- 使用者頭像與名稱卡片
- Master-Detail 分割版面
### 截圖檔名
`07_manage_virtual_user.png`
### 流程備註
這是用來配合「留言組」的假帳號資源池。
### 問題點 / 疑問點
- 頂部搜尋列旁的按鈕文字是「搜尋」而不是常見的「新增」(新增按鈕做在卡片網格第一格)。

## 11. 廠商設定
### 頁面名稱
廠商設定
### URL / 路由
`/vendor-settings` (推測)
### 頁面類型
設定 (表單)
### 主要用途
全域環境與品牌設定，包含行銷追蹤碼 (Pixel) 綁定。
### 主要區塊
- 廠商資訊 (名稱、網域、Logo)
- 時區設定
- 行銷追蹤碼 (FB / Tiktok / GTM)
- 使用者聯繫網址
### 主要按鈕與操作
- `儲存` (頂部右側)
- `登出所有裝置` (紅色大按鈕)
### 欄位與表單元素
- 文字輸入框 (廠商名稱、網域、各種 Pixel ID、Token、聯繫網址)
- 下拉選單 (時區)
### 特殊元件
- 行銷代碼的區塊設計有 `+` 新增按鈕與垃圾桶刪除按鈕 (GTM 區塊)，支援動態新增多組追蹤碼。
- 圖片預覽區 (右上角廠商 Logo)
### 截圖檔名
`11_brand_setting.png`
### 流程備註
無
### 問題點 / 疑問點
- 表單長度較長，且「登出所有裝置」按鈕放在表單正中央較為突兀，通常會放在安全性設定的獨立區塊。
### 是否屬於 MVP 必做
是
