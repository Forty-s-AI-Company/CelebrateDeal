# 幫賣後台 (Bombmy Dashboard) 頁面地圖 (Sitemap)

幫賣後台主要由一個側邊導覽列作為骨幹，架構扁平。

## 導覽列模組與層級架構

- **🏠 儀表板與數據**
  - [儀表板] `直播分析` 
    - ↳ (Modal) `查詢直播時數`

- **📦 核心資源管理**
  - [列表] `我的影片`
    - ↳ (編輯頁) 影片設定 (推測)
  - [列表] `我的商品`
    - ↳ (編輯頁) 商品設定 (推測)

- **🎥 直播與報名業務**
  - [列表] `管理報名`
    - ↳ (編輯頁) 報名腳本設定 (推測)
  - [列表] `管理直播`
    - ↳ (編輯頁) 直播間與推播設定 (推測)

- **🤖 自動化與互動工具**
  - [列表] `留言組`
  - [列表/明細] `虛擬使用者`
  - [列表] `信件模板`
  - [列表] `寄送簡訊、信箱`

- **⚙️ 平台與廠商管理**
  - [列表] `黑名單`
  - [列表] `聯盟行銷` (聯盟設定)
  - [列表] `扣點紀錄`
  - [設定] `廠商設定` (品牌資訊與 Pixel)

## 模組關聯性分析
1. **自動化營銷組合**：`虛擬使用者` 建立後，被指派到 `留言組` 的腳本中，而 `留言組` 再綁定至特定的一支 `我的影片`，最終這支影片被當作 `管理直播` 中的回放片源。
2. **流量追蹤組合**：`管理報名` 與 `聯盟行銷` 的成效，皆依賴於 `廠商設定` 內的 Pixel (FB/Tiktok) 綁定。
3. **通知系統組合**：報名成功或開播前，系統會依據 `信件模板` 的內容，產生紀錄並顯示於 `寄送簡訊、信箱` 的歷史列表中。
