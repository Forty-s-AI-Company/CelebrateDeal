# 幫賣後台 (Bombmy Dashboard) MVP 必做頁面清單

在進行產品重建或開發 MVP (最小可行性產品) 時，建議優先聚焦於「核心直播導購」與「基本轉換追蹤」相關的頁面。自動化與虛擬互動的部分可留至 Phase 2。

## 第一階段：核心 MVP (Must-Have)
這些是平台運作與廠商能順利開播、賣東西的最底限需求：

1. **廠商設定** `/vendor-settings`
   - 原因：必須要能綁定 Facebook/Tiktok Pixel 與設定基本品牌資訊，否則廠商無法追蹤轉換成效，這對於導購平台是致命傷。
2. **管理直播** `/live-streams`
   - 原因：核心業務，必須能建立與管理直播間。
3. **我的商品** `/products`
   - 原因：直播電商的核心，沒有商品就無法結帳。
4. **我的影片** `/videos`
   - 原因：若平台支援預錄影片回放 (VOD) 形式的「偽直播」，則上傳影片管理為必要功能。
5. **管理報名** `/registrations`
   - 原因：收集名單與發送通知的源頭。
6. **信件模板** `/mail-templates`
   - 原因：需要設定預約成功或開播提醒的通知內容。
7. **直播分析 (Dashboard)** `/dashboard`
   - 原因：廠商登入的第一眼，必須提供基礎成效 (如：報名數、觀看數) 的儀表板。

## 第二階段：進階/自動化營銷功能 (Nice-to-Have)
可以延後開發或作為付費加值功能的項目：

1. **留言組** & **虛擬使用者**
   - 原因：這些是為了「營造直播熱絡感」的高階行銷工具。雖然對轉換率有幫助，但並非平台無法運作的阻礙。
2. **聯盟行銷 (聯盟設定)**
   - 原因：分潤與推廣機制需要較為複雜的財務與追蹤邏輯，可待平台商業模式穩定後加入。
3. **寄送簡訊/信箱 (歷史紀錄)**
   - 原因：只要系統後台有寄送成功，初期可以先不提供給廠商看詳細的每一筆寄送 Log。
4. **黑名單**
   - 原因：初期惡意使用者較少，手動在直播間踢人即可。
5. **扣點紀錄**
   - 原因：若 MVP 採月費制或尚未導入點數扣款機制，此模組可忽略。
