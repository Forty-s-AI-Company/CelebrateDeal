# 賀成交 AI 初版 UX 與產品審查報告

本報告為「賀成交 AI」初版 (由 Codex 實作) 的深度產品巡覽與 UX 審查結果。審查角度嚴格對標成熟 SaaS 產品與真實「直播導購」業務流程的需求。

---

## 1. 總體評價

**目前這版像不像可賣產品？**
不像。這是一個擁有完整資料庫結構 (Data Model) 的 CRUD (Create, Read, Update, Delete) 系統，但距離「商業化 SaaS」還有一大段距離。

**設計感夠不夠？**
基礎的 UI 元件（如按鈕、輸入框）使用了類似 shadcn/ui 的乾淨設計，排版整齊，但整體過於扁平、單調。它看起來像是一個「給開發者自己用的管理後台」，而不是一個要賣給品牌商的付費 SaaS 產品。

**像不像幫賣這一類型產品？**
骨架有像，但血肉完全不像。它具備了幫賣的核心概念（直播間、互動腳本、角色、商品），但在「營運節奏」、「場控感」與「成交熱絡感」上完全不及格。

**最大問題是什麼？**
**「工作流的破碎與工程化」**。系統假設商家知道怎麼把這些零散的積木（腳本、角色、商品、影片）拼在一起。建立直播或腳本的過程，更像是在填寫資料庫欄位，而不是在「策劃一場會賺錢的活動」。

---

## 2. 與幫賣感的差距

1. **完全缺少「場控感」與「時間軸視覺化」**
   - 導購的靈魂在於節奏（幾分幾秒拋出優惠、假帳號何時洗版）。目前的「互動腳本」只是一長串冰冷的表單欄位（填寫秒數、選下拉選單），完全感受不到直播的起承轉合。
2. **沒有預設範本與偷吃步 (Shortcuts)**
   - 幫賣這類系統之所以好用，是因為商家「懶」。目前的系統要求商家自己新增角色、自己打留言、自己想秒數。一個成熟的 SaaS 應該一進來就有「預設的 10 種罐頭角色」跟「預設的 3 種高轉化互動腳本」可以直接套用。
3. **直播建立流程太像「設定檔」**
   - 雖然做了 Stepper（分步表單），但每一步都只是把設定欄位分開而已，完全沒有提供「預覽 (Preview)」的畫面。商家在後台勾選商品時，根本不知道前台會長什麼樣子。
4. **前台播放頁缺乏「成交急迫感」**
   - 前台（`/live/:slug`）看起來像是一個普通的一頁式購物網站（上面放影片、下面放商品列表）。沒有直播平台該有的跑馬燈、浮動商品卡、購買通知（「XXX 已購買」）等能刺激衝動消費的視覺元素。

## 3. 各頁面問題清單

### 首頁 (Dashboard) `/dashboard`
- **頁面用途**：營運數據總覽。
- **做得好的地方**：將 Onboarding checklist 放在首頁，幫助新用戶知道第一步該做什麼。
- **問題點**：過於陽春，只有三個數字方塊（觀看、報名、點擊）。沒有圖表、沒有轉化漏斗、沒有「即將到來的直播」的強烈引導。
- **建議修改方向**：加入轉化漏斗圖，並把「最近一場直播的準備進度」視覺化。

### 互動腳本 (Interaction Scripts) `/interaction-scripts/new`
- **頁面用途**：設定幾分幾秒要觸發的事件與留言。
- **做得好的地方**：事件類型（官方訊息、商品浮出、CTA）切分得很清楚。
- **問題點**：致命的難用。要在表單裡面一直按新增、手動輸入秒數 (`triggerSec`)，完全沒有拖曳 (Drag & Drop) 的時間軸概念。
- **建議修改方向**：必須改成視覺化時間軸（類似剪輯軟體或動畫軟體），水平或垂直展開，可以看到每個事件的相對位置。

### 互動角色 (Interaction Roles) `/interaction-roles/new`
- **頁面用途**：建立假帳號或官方小幫手。
- **做得好的地方**：有頭像、名稱、語氣設定。
- **問題點**：必須從零開始自己建。沒有「批次匯入」或「套用角色包」的功能。
- **建議修改方向**：提供系統內建角色庫（例如：鐵粉、好奇寶寶、催單小幫手），讓商家一鍵加入 5 個預設角色。

### 直播間建立 `/lives/new`
- **頁面用途**：把影片、商品、腳本綁定成一場直播。
- **做得好的地方**：採用 8 個步驟的 Stepper，避免一頁長表單的壓力。
- **問題點**：在「綁定商品」與「綁定腳本」的步驟中，完全沒有視覺反饋。
- **建議修改方向**：在最後一步「發布」之前，應該要有一個「前台預覽 (Mockup)」畫面，讓商家看到最終合體後的樣子。

## 4. 關鍵模組深度評論

### 虛擬使用者 / 互動角色
- **現在的問題**：純資料庫列表。使用者必須手動輸入角色名稱、上傳頭像，非常耗時。
- **為什麼不人性化**：商家要的是「氣氛」，不是「建立人事資料」。
- **應補哪些互動**：新增一個「從範本庫挑選」的按鈕，點擊後會彈出 20 個已經設定好頭像與語氣的虛擬人物，勾選後即可批量加入。

### 留言組 / 互動腳本
- **現在的問題**：這只是個「擁有 triggerSec 欄位的 Array Form」，完全沒有腳本的感覺。
- **為什麼不人性化**：當腳本超過 10 句留言時，這種表單會長到難以管理，且無法直觀看出 10 分鐘到 15 分鐘之間的密集度。
- **更像幫賣應該怎麼做**：實作一個簡易的 Timeline UI。左邊是時間軸，右邊是留言框；可以按住留言卡片上下拖曳來改變觸發時間。
- **預設內容**：新增腳本時，直接提供「30 分鐘服飾銷售腳本」、「1 小時保養品腳本」讓用戶一鍵套用修改。

### 直播間建立流程
- **現在的問題**：缺乏「模板化」與「預覽」。
- **更像幫賣應該怎麼做**：第一步不應該是填名稱，而是「選擇直播版型/目標（例如：新品發表、清倉拍賣）」。不同的版型會自動幫你帶入預設的腳本與 CTA。設定過程中，右側可以常駐一個「手機版型預覽圖」，隨著左側設定即時更新。

### 前台播放頁
- **現在的問題**：(參考截圖 `24_live_frontend.png`) 這只是個「上面放影片、下面放 Tab」的普通網頁。毫無直播的衝動感。
- **更像幫賣應該怎麼做**：影片必須滿版或佔據主要視覺；商品卡應該以「浮水印 / 彈出式卡片」的方式覆蓋在影片邊緣（特別是在手機版）。聊天室應該要自動往上捲動，並加入虛擬人物的顏色標籤與頭像。

## 5. 必改清單（高優先）

1. **問題**：互動腳本編輯器是純表單，極度難用。
   **影響**：商家會放棄設定互動，失去產品核心價值。
   **建議改法**：立刻重構 `interaction-script-form.tsx`。改為垂直時間軸 UI，事件用卡片呈現，支援展開/收合，並優化時間輸入方式（例如：支援輸入 `01:30` 而不是只能輸入 `90` 秒）。

2. **問題**：前台播放頁 (`/live/:slug`) 缺乏導購感與直播 UI。
   **影響**：消費者轉化率低，產品顯得廉價。
   **建議改法**：重構 `live-playback.tsx`。參考 TikTok Live 或 Shopee Live 的介面：影片置頂/滿版、聊天室為半透明覆蓋在影片下方或右側、商品以 Pop-up 卡片形式在指定秒數時彈出，而不是呆板地放在下面的 Tab 裡。

3. **問題**：沒有預設的虛擬角色與腳本範本。
   **影響**：Onboarding 阻力極大，冷啟動困難。
   **建議改法**：在 Database seed 中，或在建立流程的 UI 裡，加入「一鍵匯入預設角色庫 (10個角色)」與「一鍵匯入罐頭腳本」的功能。

---

## 6. 可補強清單（中優先）

- **Dashboard 豐富化**：加入漏斗圖（進站 -> 點擊 -> 報名），以及「近期即將開播」的倒數計時卡片。
- **即時預覽 (Live Preview)**：在建立直播的 Stepper 中，加入右側或彈窗的前台預覽畫面。
- **大量操作 (Bulk Actions)**：互動角色列表與腳本列表應支援批次刪除、複製。

## 7. 給 Codex 的具體修改指令

*(請直接複製以下文字貼給 Codex)*

> 「Codex，目前的初版系統架構與資料庫很完整，但整體的 UX 與 UI 太像一個『給工程師用的資料庫後台』，完全缺乏『SaaS 產品的設計感』與『直播導購的營運感』。
> 
> 請你先放下新功能的開發，優先進行以下重構：
> 1. **重構互動腳本 (`interaction-script-form.tsx`)**：這不能只是個 `triggerSec` 的輸入表單。必須將它改成『視覺化時間軸』。時間輸入必須支援 `MM:SS` 格式，事件必須像卡片一樣排列，並且要預載一組『常見導購節奏』的預設範本供商家修改。
> 2. **重製前台播放頁 (`live-playback.tsx`)**：現在的版本太像一般電商商品頁。請參考 Shopee Live 或 TikTok，影片畫面要放大，聊天室要能自動捲動且具備明顯的虛擬人物標籤，『商品浮出』事件觸發時，商品卡必須以 Pop-up 的動畫形式蓋在畫面上刺激購買。
> 3. **優化虛擬角色與流程體驗**：商家不想手動建假帳號。請在建立角色的地方加入『從系統庫一鍵匯入 10 個常用人設』的功能。另外，直播建立的 Stepper 在最後一步必須給出『前台預覽圖』。
> 
> 請不要只補程式碼邏輯，請務必把 Tailwind 的版面、漸層、陰影、卡片層級做出來，讓它看起來像一個每個月收費 $99 美金的高級 SaaS 產品！」

## 評分總結

- **視覺設計感**：4/10 (乾淨但不高級，過於扁平、空白太多，缺乏 SaaS 質感)
- **SaaS 成熟度**：5/10 (有完整的選單與權限架構，但缺乏 Onboarding 與 Dashboard 洞察)
- **人性化程度**：3/10 (表單過度依賴手動輸入，缺乏預設值、範本與批次操作)
- **幫賣感 / 導購感**：2/10 (完全像一般 CMS 後台，感受不到在操作一場刺激的銷售活動)
- **直播建立流程**：6/10 (Stepper 邏輯清晰，但缺乏預覽與版型選擇)
- **虛擬使用者可用性**：3/10 (只能從零手動建立，完全沒有資源池的概念)
- **留言組可用性**：2/10 (用傳統表單輸入秒數與對話，不符合時間軸編排的人類直覺)
- **前台成交頁完成度**：3/10 (只是個帶有 Tab 的影片播放頁，缺乏即時互動與衝動消費的視覺刺激)
