# Design Review Checklist

- Page has one clear primary task and action.
- Heading, supporting text, data, labels, and metadata have distinct hierarchy.
- Shared tokens cover surface, text, border, primary, CTA, success, warning, danger, info, focus, and disabled.
- Table supports scanning, search/filter where needed, pagination, empty and error states.
- Form labels describe user concepts; errors explain recovery.
- Focus is visible; selected/current/live regions expose semantics.
- Text and controls meet contrast; orange background is dark enough for white text.
- Motion respects `prefers-reduced-motion`.
- 390/768/1280/1440 layouts do not overlap, clip, or force unreadable columns.
- Marketing celebration does not leak into routine admin work.
